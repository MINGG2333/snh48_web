# KB_QA 参数配置说明

## 中央配置位置

所有问答参数的默认值集中管理在：

```
transcript_analyze/kb_qa/config.py
```

**只需修改这一个文件**，即可同时影响 CLI 和 Web 服务两端：

- `transcript_analyze/kb_qa/cli.py`（命令行问答，如 `python run_kb_qa.py ask`）
- `website/qa_api/router.py`（Web 服务问答，如网站页面上的提问）

## 参数一览

### 检索阶段参数

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| `vector_top_k` | `20` | 向量检索实际使用的 top_k。从 survey 结果中取前 N 条用于后续分析。值越大检索越全面，但更多片段的分析/合成会增加 LLM API 消耗。 |
| `bm25_top_k` | `20` | BM25 检索实际使用的 top_k。从 survey 结果中取前 N 条（仅作为向量的少量补充）。 |
| `vector_score_threshold` | `0.31` | 向量检索相关性阈值 [0-1]。低于此分数的结果被过滤，不参与后续合并。值越低召回越多但噪声也可能增多。 |
| `bm25_score_threshold` | `11.0` | BM25 检索相关性阈值。低于此分数的结果被过滤。 |

### Survey（调查）阶段参数

> Survey 是用于**全面性评估**的额外检索步骤，不参与实际分析。

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| `vector_survey_top_k` | `1000` | 向量检索 survey 阶段的 top_k。仅用于计算「预估相关段总数」。部署时可根据服务器内存调整，值越大预估越准。 |
| `bm25_survey_top_k` | `1000` | BM25 检索 survey 阶段的 top_k。 |

**为什么需要 survey 阶段？**

对于高频词问题（如 "陈嘉仪"、"顺顺"），数据库中相关的片段可能有数千条。如果全部检索并分析，耗时和 API 消耗会非常高。因此采用两阶段策略：

1. **Survey**：以较大 top_k（1000）检索，找到**所有**超过相关性阈值的片段，计算预估总量
2. **实际使用**：只取前 `vector_top_k`/`bm25_top_k` 条（默认各 20 条）用于后续分析

这样可以用少量片段回答大部分问题，同时通过**全面性比率**告知用户回答的覆盖程度。

### 上下文扩展参数

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| `context_window` | `10` | 合并后的基础段前后各扩展多少条**同直播同来源类型**的片段作为上下文。值越大每个片段的背景信息越多，但候选总量膨胀。对于高频词问题，太大的窗口可能导致候选数激增。 |

### 分析阶段参数

> 当前已跳过 LLM 有用性分析（方案 C），此参数暂不生效。

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| `analysis_batch_size` | `100` | 每批交给 LLM 分析的候选段数量。调大减少 API 调用次数，但单次上下文太长可能影响准确率。 |

### 合成阶段参数

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| `synthesis_context_window` | `6` | 合成时每个片段在 prompt 中前后各取 N 条同直播同类片段作为局部上下文。 |
| `synthesis_batch_trigger_count` | `100` | 有用段超过此数量时触发分批合成（当前全部候选保留，未真正启用）。 |
| `synthesis_batch_size` | `50` | 分批合成时每批的大小。 |

## 调参建议

### 想让回答更全面
- 增大 `vector_top_k` 和 `bm25_top_k`（如各 100）
- 减小 `vector_score_threshold`（如 0.25）
- 注意：这会显著增加 LLM API 调用量和处理时间

### 想让回答更快/更省
- 减小 `vector_top_k` 和 `bm25_top_k`（如各 10）
- 适当提高 `vector_score_threshold`（如 0.35）
- 注意：可能会降低回答的全面性

### 应对高频词问题（如 "陈嘉仪"）
- 保持 `vector_top_k`/`bm25_top_k` 较小（如各 20）
- 提高 `bm25_score_threshold`（如 15.0），减少 BM25 噪声
- 页面会显示全面性比率，低比率时可通过邮箱请求更全面的回答

## 数据流图

```
用户提问
  │
  ▼
向量查询改写 ──→ 向量Survey (1000条) ──→ 取前 vector_top_k 条 ──┐
                                                               │
BM25查询改写 ──→ BM25 Survey (1000条) ──→ 取前 bm25_top_k 条  ─┤
                                                               │
                                                               ▼
                                                   合并（向量全保留 + BM25补充）
                                                               │
                                                               ▼
                                                   上下文扩展 (context_window)
                                                               │
                                                               ▼
                                                   跳过分析（全部视为有用）
                                                               │
                                                               ▼
                                                   按直播分组合成 → 最终答案
                                                               │
                                                               ▼
                                             全面性评估（展示给用户）
```
