# 记忆页面产品记录

> 更新日期：2026-07-06

本页记录“记忆”页面的长期产品原则、数据边界和第一版实现约定。这里的“记忆”不是榜单，也不是贡献排行，而是记录“被喜欢过的证据”。

## 初衷

- 这个网站记录“被喜欢过的证据”，不是记录“谁更喜欢她”。
- 它记录的不是“谁更爱她”，而是“这些喜欢曾经真实存在过”。
- 每条记录叫“记忆”，不叫“贡献”。
- 页面不做粉丝排名，不做拉踩比较；统计只用于理解资料规模和整理进度。
- 所有喜欢陈嘉仪的人都可以有参与感：公开互动、虚拟礼物、内容创作、幕后整理、集体记忆和粉丝自述都可以被记录。

## 记忆分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 小偶像回应粉丝 | 陈嘉仪对粉丝的公开或半公开回应 | 口袋48房间回复、直播念到名字、直播回应弹幕、微博评论回复、B站/抖音互动、线下活动中的公开回应 |
| 粉丝表达喜欢 | 粉丝主动表达喜欢或留下作品 | 直播礼物、房间礼物、生日祝福、长文、repo、视频剪辑、二创、翻唱、剪影、表情包、安利帖、数据整理、字幕、搬运、投稿 |
| 粉丝为她做的事 | 粉丝或应援会为她持续维护、组织或制作的内容 | 应援会活动、生日项目、线下布置、物料设计、打投组织、行程整理、直播切片、超话维护、新粉科普、公益或纪念项目 |
| 共同记忆 | 属于她和粉丝群体的阶段、事件或梗 | 某次直播名场面、某场公演、某个梗、一次集体祝福、一次大家一起陪她熬过去的时期 |
| 私密或半私密互动 | 粉丝愿意自述、但不适合强制公开证据或细节的记忆 | “某日线下活动中得到了鼓励”；证据可选，公开内容允许模糊化 |

## 时间范围

第一版默认从 `2025-09-01` 开始整理。这个日期用于覆盖陈嘉仪出道初期到现在的资料；如果后续确认更准确的出道日期或需要补录更早准备期资料，可以调整导入脚本的 `--start-date`。

## 隐私和展示边界

- 小房间消息可以纳入记忆页面，但页面本身应设置访问密码。
- 回礼物原文可以展示；它来自口袋48房间消息等既有数据源。
- 私下互动以粉丝自愿登记为前提，不强迫公开证据，不强迫写细节。
- 前台可以展示平台昵称和平台名称。
- 平台 ID、口袋48 sender_id、微博 uid、B站 mid 等身份标识只保留在后台数据中，用于去重、核对和后续合并身份；默认不展示给普通访问者。
- 来源链接、截图说明、证据摘要应尽量保留，但不要要求粉丝为了登记而暴露私密聊天、身份证明或不愿公开的细节。

## 审核和确认模型

第一版数据状态分成两层：

| 层级 | 字段 | 说明 |
|------|------|------|
| 展示审核 | `audit_status`、`visibility` | 自动基础检查通过的记忆可以展示；疑似恶意、辱骂、广告或格式异常的内容进入人工审核，不直接公开 |
| 事实确认 | `confirmation_status` | 公开展示后仍可处于“待确认”；应援会确认后显示“应援会确认”，陈嘉仪本人确认后显示“本人确认” |

密码模式：

- 普通访问/提交：`MEMORIES_VIEW_PASSWORD`
- 应援会模式：`MEMORIES_FANCLUB_PASSWORD`
- 本人模式：`MEMORIES_IDOL_PASSWORD`

第一版只做基础规则审核和人工审核入口，不在网站服务进程里直接调用 Codex。后续可以把待审记忆写入 fan-hub 的 Codex 自动处理队列，再把 Codex 判定结果合并回 `memories.json`。

## 数据文件

运行数据文件：

```text
website/data/memories/memories.json
```

仓库只提交格式示例：

```text
website/data/memories/memories.example.json
```

`memories.json` 是运行数据，不进入 Git。腾讯云作为写入源时，阿里云通过现有“阿里云主动从腾讯云拉取”的同步链路获取副本。若未来希望两台服务器都开放提交，需要先设计双向合并或统一写入 API，不能让两台服务器各自写本地文件。

## 第一版导入命令

从 fan-hub 现有数据生成初始记忆：

```bash
python3 script/build_memories_seed.py
```

常用参数：

```bash
python3 script/build_memories_seed.py --start-date 2025-09-01 --gift-replies-limit 160 --live-gifts-limit 160 --events-limit 120
```

导入脚本会尽量使用稳定来源 ID 生成记忆 ID，并保留已有人工确认状态。
