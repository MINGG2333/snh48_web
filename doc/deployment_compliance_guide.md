## 四、域名与 HTTPS

本网站已购买域名 `cjy.plus`，需要完成以下配置步骤（在服务器上操作）。

### 4.1 ✅ 域名解析（已完成）

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | @ | 服务器公网 IP |
| A | www | 服务器公网 IP |

### 4.2 配置 Nginx（需在服务器上操作）

登录服务器后执行：

```bash
# 1. 编辑 nginx.conf，替换域名
sed -i 's/your-domain.com/cjy.plus/g' deploy/nginx.conf

# 2. 复制配置到 nginx
sudo cp deploy/nginx.conf /etc/nginx/conf.d/snh48.conf

# 3. 测试配置
sudo nginx -t

# 4. 重载 nginx
sudo systemctl reload nginx
```

### 4.3 申请 SSL 证书（需在服务器上操作）

```bash
# 先停止 nginx（80 端口需要被 certbot 占用）
sudo systemctl stop nginx

# 申请证书
sudo certbot certonly --standalone -d cjy.plus -d www.cjy.plus

# 重新启动 nginx
sudo systemctl start nginx

# 设置自动续期
sudo certbot renew --dry-run
```

## 五、安全评估填报指南

> ⚠️ **本网站因含有 AI 问答功能（生成式人工智能服务），被系统判定具有舆论属性，需要进行安全评估。**

### 5.1 评估填报步骤

1. 登录 **全国互联网安全管理服务平台**
2. 找到您的网站申请 → 找到"安全评估"入口
3. 选择 **自评估**
4. 逐项填写：

| 表单问题 | 答案参考文件 | 对应章节 |
|----------|-------------|---------|
| 开展评估情况 | `security_assessment_qa.txt` | 一 |
| 评估方法 | `security_assessment_qa.txt` | 二 |
| 安全管理负责人 | `security_assessment_qa.txt` | 三（需填写您本人信息） |
| 用户真实身份核验及注册信息留存 | `security_assessment_qa.txt` | 四 |
| 日志信息留存措施 | `security_assessment_qa.txt` | 五 |
| 违法有害信息防范处置 | `security_assessment_qa.txt` | 六 |
| 个人信息保护及技术措施 | `security_assessment_qa.txt` | 七 |
| 投诉举报机制 → **选择"是"** | `security_assessment_qa.txt` | 八 |
| 为监管部门提供技术协助的机制 | `security_assessment_qa.txt` | 九 |

### 5.2 注意事项

- 安全管理负责人的信息请填写您本人的真实信息
- 建议先在文本编辑器中填好后再复制到平台
- 填写完成后先保存草稿，核对无误再提交
- 如有附件上传入口，可上传 `security_assessment_report.md` 作为补充材料

## 八、法律义务提示

根据现行法规，请务必注意：

1. **配合监管义务：** 如收到网信办/公安部门的通知，需在要求时限内配合调查和处置
