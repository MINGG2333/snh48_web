## 四、网站功能概述

本网站是一个个人粉丝站点，主要功能如下：

### 4.1 信息展示功能
- **首页：** 全屏动态背景展示，滚动显示关键词/标语
- **关于页：** 展示项目目标和开发状态

### 4.2 AI 智能问答功能（生成式人工智能服务）
- 基于 LLM（大语言模型）的知识库问答系统
- 用户可输入问题，AI 根据预先构建的知识库生成回答
- 回答附带引用来源（citation），可追溯信息来源
- 需要密码验证才能使用

### 4.3 背景词管理功能
- 管理员可通过密码验证后管理首页滚动显示的文本内容
- 普通用户只能查看，不能修改

## 五、舆论属性与社会动员能力分析

### 5.1 舆论属性分析

本网站**具有舆论属性**，具体体现在：

| 功能 | 是否具有舆论属性 | 说明 |
|------|-----------------|------|
| AI 智能问答 | **是** | 基于生成式人工智能技术，可根据用户提问生成文本内容，具有信息传播能力 |
| 信息展示 | 否 | 仅展示静态信息，无用户评论、互动功能 |
| 背景词管理 | 否 | 仅管理员可管理，普通用户无发布权限 |

### 5.2 社会动员能力分析

本网站**不具有社会动员能力**，理由如下：

- 网站为个人粉丝站点，非公共舆论平台
- **未开设论坛、评论区、群组等互动功能**
- 用户之间无法互相交流或组织活动
- AI 问答功能需密码验证，访问受限
- 网站未设置任何动员、号召、聚集功能

### 5.3 综合评估结论

> **结论：** 本网站具有舆论属性（因提供生成式人工智能服务），但不具有社会动员能力。根据《具有舆论属性或社会动员能力的互联网信息服务安全评估规定》，需要进行安全评估。

## 六、安全管理制度与措施

### 6.1 安全管理负责人

已指定专人担任安全管理负责人，负责：
- 网站安全管理制度制定和执行
- 网络安全事件应急响应
- 内容安全审核
- 用户投诉举报处理

### 6.2 用户真实身份核验及注册信息留存措施

| 措施 | 说明 |
|------|------|
| 身份核验方式 | AI 问答功能采用**密码验证**方式，用户需输入网站密码才能使用。密码由管理员设置，仅向可信任用户告知。 |
| 注册信息留存 | 本网站**不要求用户注册登录**，不收集用户姓名、身份证号、手机号等身份信息。 |
| 临时标识 | 使用浏览器 `sessionStorage` 生成随机 client_id（如 `user_a1b2c3d4_1a2b3c`），仅用于区分用户会话和限速管理。 |

### 6.3 日志信息留存措施

| 日志类型 | 记录内容 | 留存方式 | 留存期限 |
|----------|---------|---------|---------|
| AI 问答交互日志 | 用户问题、AI回答、引用来源、客户端ID、时间戳 | JSONL 文件（/interaction_logs/session_*/） | 每次服务器启动时自动备份旧日志 |
| LLM 调用日志 | 用户ID、请求内容、模型响应、Token 使用量、成功/失败状态 | 按用户分文件存储 | 随交互日志一起备份 |
| API 错误日志 | 客户端ID、端点、错误信息、时间戳 | 合并日志文件 | 随交互日志一起备份 |
| Nginx 访问日志 | 来源IP、请求路径、HTTP状态码、User-Agent | Nginx access_log | 默认保留 |
| 邮箱收集记录 | 用户自愿提交的邮箱（仅用于超时结果通知） | JSONL 文件存储 | 任务完成后可删除 |

**日志信息包含：**
- 用户标识（随机client_id）
- 操作时间（timestamp）
- 操作类型（提问/查看/密码验证）
- 网络源地址（IP；经 Nginx 设置 `X-Real-IP` / `X-Forwarded-For`，后端仅在请求来自显式配置的可信代理时采信代理头，默认仅信任本机 Nginx）
- 目标地址（API 端点）
- 客户端硬件特征（通过 User-Agent 和请求头获取，不主动采集指纹）

### 6.4 违法有害信息防范处置措施

#### 6.4.1 输入过滤与输出安全（前端+后端双层防护）

| 层级 | 措施 | 说明 |
|------|------|------|
| 前端 | 字符白名单过滤 | 仅允许中文、英文字母、数字和常用标点符号，拦截特殊符号/XSS攻击向量 |
| 前端 | 长度限制 | 有效字符不超过20字 |
| 前端 | 输出转义 | QA 答案、引用、时光轴标题/描述等动态内容进入 HTML 前进行转义，避免脚本注入 |
| 前端 | URL/图标白名单 | 外部链接、图片地址仅允许 http/https 或同源相对路径，图标类名进行格式校验 |
| 后端 | 字符白名单验证 | 使用正则表达式二次验证 |
| 后端 | 长度限制 | 使用`MAX_QUESTION_LENGTH = 20`后端验证 |
| 网关 | CSP 与安全响应头 | Nginx 配置 CSP、HSTS、X-Frame-Options、X-Content-Type-Options、Referrer-Policy，降低浏览器侧攻击面 |

#### 6.4.2 AI 内容安全管控

在所有 LLM 提示词（Prompt）中嵌入**内容安全规则**：

```
【内容安全规则】你的回答会被公开发布在个人网站上：
1) 必须符合中国法律法规，坚持社会主义核心价值观。
2) 不得生成任何违法内容（包括但不限于：煽动颠覆国家政权、
   危害国家安全、恐怖主义、民族仇恨、淫秽色情、虚假有害信息等）。
3) 如果用户问题涉及违法内容，必须明确拒绝回答。
4) 回答需客观、负责任，避免对主播造成误导或负面形象。
```

此规则嵌入在所有 LLM 调用环节：
- 直接问答 Prompt
- 分组/批次合成 Prompt
- 最终答案合成 Prompt
- 答案合并 Prompt

#### 6.4.3 内容标识

- **页面级标识：** "本服务使用生成式人工智能技术，生成内容仅供参考，不代表陈嘉仪本人立场，请理性看待。"
- **答案级标识：** "以上内容由人工智能（AI）生成，仅供参考，不代表陈嘉仪本人立场。请结合其他信息源自行判断。"

#### 6.4.4 访问控制

- **密码保护：** AI 问答功能需密码验证才能使用
- **异步结果保护：** `/api/qa/ask-async/{task_id}` 结果轮询需密码验证，并校验 `X-Client-Id` 与任务所属会话一致，同时要求提交任务时返回的一次性 `poll_token`
- **IP 限速：** 每个 IP 每 60 秒最多 5 次提问
- **用户冷却：** 两次提问之间至少间隔 30 秒
- **每日配额：** 每用户每天最多 50 次提问
- **并发限制：** 每用户最多同时处理 2 个任务
- **密码暴力破解防护：** 每个 IP 每 300 秒最多 10 次密码尝试
- **公开端点防滥用：** 追踪事件、投诉提交、邮箱提交、余额查询、背景词/观察页登录尝试均设置 IP 级限速
- **余额接口缓存：** 余额查询成功结果短期缓存，减少对第三方 API 的重复调用

### 6.5 个人信息保护技术措施

| 措施 | 说明 |
|------|------|
| 最小化收集 | 不强制收集用户个人信息，不要求注册登录 |
| 匿名化标识 | 使用随机 client_id 代替真实身份 |
| 邮箱选择性收集 | 仅在用户主动提交时收集，有明确用途说明 |
| 数据本地化 | 所有数据存储在中国服务器 |
| 第三方LLM服务管理 | AI问答功能调用境内合法备案的LLM服务商（DeepSeek），数据传输使用HTTPS加密，服务商已按《生成式人工智能服务管理暂行办法》要求履行算法备案等义务 |
| 访问控制 | 敏感管理功能需密码验证 |
| 来源 IP 可信化 | Nginx 设置真实来源 IP，后端仅在请求来自显式配置的可信代理时采信代理头，默认仅信任本机 Nginx，避免客户端或同内网机器伪造 `X-Forwarded-For` |
| 安全 Cookie | HTTPS 生产环境启用 `Secure` Cookie，管理 Cookie 不通过明文 HTTP 发送 |
| 日志脱敏 | 日志中不以明文存储用户敏感信息 |
| 定期清理 | 日志采用会话制管理，定期备份并支持清理 |

### 6.6 防止违法有害信息传播和防范社会动员功能失控风险的技术措施

由于本网站**不提供用户间互动功能**（无评论、无论坛、无群组），社会动员功能失控风险极低。已采取以下措施：

1. **单向信息流：** 用户只能向 AI 提问获取信息，无法在网站内与其他用户互动
2. **站内无转发/评论功能：** 网站不提供站内转发、评论、私信等互动功能
3. **保存为图片功能：** AI 问答页面提供"保存为图片"（截图下载）功能，该功能为客户端本地操作，网站不介入转发或传播环节。所有 AI 回答在页面和截图中均附带"AI 生成内容仅供参考"的免责标识
4. **无用户生成内容（UGC）：** 用户不能发布内容供网站内其他用户查看
5. **密码访问控制：** 限制 AI 功能的访问范围
6. **速率限制：** 防止 API 滥用
7. **内容安全 Prompt：** AI 拒绝生成违法违规内容
8. **输入过滤：** 拦截恶意输入和注入攻击
9. **输出转义：** 动态内容展示前进行 HTML 转义，降低恶意脚本传播风险
10. **浏览器安全策略：** 通过 CSP、HSTS 等安全头限制外部资源加载和降级访问

### 6.7 投诉举报机制

- **入口：** 网站设有"投诉与举报"页面（`/complaint`），用户可在线提交投诉
- **处理流程：**
  1. 用户通过表单提交投诉举报内容
  2. 系统自动记录并通知管理员
  3. 管理员在 24 小时内受理
  4. 在 7 个工作日内完成处理并反馈
- **反馈渠道：** 用户提交后可获取受理编号，通过邮箱查询处理进度

## 八、安全评估结论

### 8.1 评估结果

| 日志数据安全 | 低 | 日志存储在本地服务器，访问需管理员权限 | ✅ 已完成 |

### 8.3 最终评估结论

> **结论：** 本网站的安全管理制度和技术措施**符合**《具有舆论属性或社会动员能力的互联网信息服务安全评估规定》的要求，安全风险可控，建议通过安全评估。
