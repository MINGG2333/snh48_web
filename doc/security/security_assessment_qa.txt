四、用户真实身份核验及注册信息留存措施
────────────────────────────────────────────────────────────────

说明：请描述网站对用户真实身份进行核验的方式，以及注册信息
的留存措施。

▶ 填报内容：

（一）身份核验方式
本网站不要求用户注册登录，不收集用户的姓名、身份证号、手机
号、住址等个人信息。AI 问答功能采用密码验证方式进行访问控
制，用户需输入网站管理员预设的密码才能使用该功能。密码通过
安全的 .env 环境变量配置，不在代码中硬编码，不对外公开，仅
向可信任的用户告知。密码验证服务端使用速率限制（每 IP 每
300 秒最多 10 次尝试）以防止暴力破解。

（二）注册信息留存
由于本网站不设用户注册功能，因此不收集、不留存任何用户注册
信息。这是本网站"最小化信息收集"原则的体现，符合《个人信
息保护法》关于"处理个人信息应当具有明确、合理的目的，并与
处理目的直接相关"的要求。

（三）临时会话标识
为了区分用户会话和实现速率限制，网站使用浏览器
sessionStorage 生成随机 client_id（格式如
"user_a1b2c3d4_1a2b3c"），该标识仅临时存储在用户浏览器中，
关闭浏览器标签页后自动清除。此标识仅用于会话管理的技术目的，
不与任何真实身份信息关联。

▶ 填报内容：

（一）日志记录内容
本网站记录以下日志信息：

1. AI 问答交互日志（JSONL 文件）：
   - 用户标识（随机 client_id，不关联真实身份）
   - 操作时间（ISO 8601 格式时间戳）
   - 操作类型（提问、查看结果、密码验证）
   - 用户提问内容
   - AI 生成回答内容
   - 引用来源（citation）
   - 网络源地址（通过 X-Forwarded-For 获取的 IP 地址）
   - 目标地址（API 端点路径）
   - 请求处理状态（成功/失败）

2. LLM 调用日志（按用户分文件存储）：
   - 用户标识
   - 请求内容
   - 模型响应
   - Token 使用量
   - API 调用耗时
   - 成功/失败状态

   - 端点路径
   - 错误类型和错误信息
   - 时间戳

4. Nginx 访问日志（服务器层面）：
   - 来源 IP 地址
   - 请求时间
   - 请求方法（GET/POST）
   - 请求路径
   - HTTP 状态码
   - User-Agent（客户端浏览器信息）
   - 来源页面（Referer）

5. 邮箱收集记录（JSONL 文件）：
   - 用户自愿提交的邮箱地址（仅用于 AI 问答超时结果通知）
   - 关联的随机 client_id
   - 提交时间

（二）日志信息覆盖范围
- 用户标识：√ 已记录（随机 client_id）
- 操作时间：√ 已记录（timestamp）
- 操作类型：√ 已记录（提问/查看/密码验证）
- 网络源地址：√ 已记录（IP，通过 X-Forwarded-For）
- 目标地址：√ 已记录（API 端点）
- 网络源端口：Nginx 层可记录（默认保留 HTTP 连接信息）
- 客户端硬件特征：√ 已记录（User-Agent，通过请求头获取，
  不主动采集设备指纹或硬件信息）
- 用户发布信息：√ 已记录（AI 问答的提问和回答内容）

（三）日志留存方式
- 存储位置：服务器本地文件系统
- 存储格式：JSONL（每行一条 JSON 记录）
- 目录结构：/interaction_logs/session_YYYYMMDD_HHMMSS/
- 访问权限：仅管理员可通过 SSH 登录服务器访问

（四）日志留存期限
- 每次服务器启动时，旧的会话日志自动备份到 logs_backup 目录
- 备份日志保留至少 6 个月
- 历史备份由管理员定期审查和清理
- 邮箱地址在相关任务完成后可删除

（五）日志安全措施
- 日志文件仅限管理员通过 SSH 访问
- 日志中不以明文存储用户密码或其他敏感信息
- 客户端 ID 为随机生成的匿名标识，无法反查用户真实身份
- 服务器部署在中国大陆境内

六、违法有害信息防范处置措施
────────────────────────────────────────────────────────────────

说明：请描述网站对用户账号和通讯群组名称、昵称、简介、备注、
标识，信息发布、转发、评论和通讯群组等服务功能中违法有害信
息的防范处置和有关记录保存措施。

▶ 填报内容：

（一）本网站不提供的功能
特别说明：本网站是个人粉丝站点，具有以下功能特性：

   √   开放的互动功能（论坛、评论区、群组等）：本网站不提供
   √   用户账号系统：本网站不要求用户注册
   √   用户发布/转发/评论功能：本网站不提供
   √   通讯群组功能：本网站不提供
   √   用户间私信/交流功能：本网站不提供
   √   用户昵称/简介/备注/标识设置：本网站不提供

因此，在"用户账号和通讯群组名称、昵称、简介、备注、标识，
信息发布、转发、评论和通讯群组等服务功能"方面，本网站不存
在上述功能，相应的违法有害信息风险极低。

（二）已实施的违法有害信息防范措施

1. 输入过滤（前端 + 后端双层过滤）
   - 前端过滤：仅允许中文、英文字母、数字和常用标点符号，
     拦截 XSS 攻击向量和特殊字符
   - 前端长度限制：有效字符不超过 20 字
   - 后端验证：使用正则表达式二次校验，确认字符合法性
   - 后端长度限制：MAX_QUESTION_LENGTH = 20

2. AI 内容安全管控（Prompt 安全规则）
   在所有 LLM 调用环节嵌入内容安全规则：
   ```
   【内容安全规则】你的回答会被公开发布在个人网站上：
   1) 必须符合中国法律法规，坚持社会主义核心价值观。
   2) 不得生成任何违法内容（包括但不限于：煽动颠覆国家政权、
      危害国家安全、恐怖主义、民族仇恨、淫秽色情、虚假有害
      信息等）。
   3) 如果用户问题涉及违法内容，必须明确拒绝回答。
   4) 回答需客观、负责任，避免对主播造成误导或负面形象。
   ```
   嵌入环节包括：直接问答 Prompt、分组/批次合成 Prompt、
   最终答案合成 Prompt、答案合并 Prompt

3. 内容标识
   - 页面级标识（AI 问答页面顶部）：
     "本服务使用生成式人工智能技术，生成内容仅供参考，不代
     表陈嘉仪本人立场，请理性看待。"
   - 答案级标识（AI 回答底部）：
     "以上内容由人工智能（AI）生成，仅供参考，不代表陈嘉仪
     本人立场。请结合其他信息源自行判断。"

4. 访问控制
   - 密码验证：AI 问答功能需密码才能使用
   - IP 限速：每 IP 每 60 秒最多 5 次提问
   - 用户冷却：两次提问至少间隔 30 秒
   - 每日配额：每用户每天最多 50 次提问
   - 并发限制：每用户最多同时处理 2 个任务
   - 暴力破解防护：每 IP 每 300 秒最多 10 次密码尝试

（三）相关记录保存
- AI 问答交互记录（包括用户提问、AI 回答、时间戳）均被记
  录在 JSONL 日志文件中，支持追溯审查
- 日志记录包含完整的请求和响应内容，可在发现违法信息时追
  溯到具体会话
- 日志文件按会话周期管理，定期备份，至少保留 6 个月

七、个人信息保护及技术措施
────────────────────────────────────────────────────────────────

说明：请描述网站的个人信息保护措施，以及防范违法有害信息传
播扩展、社会动员功能失控风险的技术措施。

▶ 填报内容：

（一）个人信息保护措施

1. 最小化收集原则
   - 不强制收集任何个人信息
   - 不要求用户注册或提供真实姓名、身份证号、手机号、住址
     等
   - 仅收集提供服务所必需的最少信息（随机 client_id、IP、
     交互内容）

2. 匿名化处理
   - 使用随机 client_id 代替真实身份
   - client_id 由浏览器 sessionStorage 生成，与服务端不共享
     用户真实身份信息
   - 日志中存储的为用户匿名标识，无法关联到特定自然人

3. 用户自主选择
   - 邮箱地址为可选提供，仅在用户主动提交时收集
   - 收集时明确告知用途（仅用于 AI 问答处理结果通知）

4. 数据安全存储与第三方服务管理
   - 所有数据存储在中国大陆服务器
   - 日志文件仅管理员可通过 SSH 访问
   - AI 问答功能使用境内合法 LLM 服务提供商（DeepSeek，境内备案企业），
     用户提问内容在加密传输下发送至其 API 接口进行处理，符合《生成式
     人工智能服务管理暂行办法》关于生成式人工智能服务提供者的管理要求
   - 与 DeepSeek 之间的数据传输采用 HTTPS 加密，不在传输过程中留存明
     文用户数据于中间环节
   - 不将用户数据用于任何其他第三方数据分析、广告推荐或商业用途
   - 不与 LLM 服务提供商之外的任何第三方共享用户数据

5. 数据留存期限
   - 采用会话制管理，服务器重启时自动备份旧日志
   - 备份日志保留至少 6 个月
   - 邮箱地址在任务完成后可应要求删除
   - 用户可通过投诉举报渠道要求删除个人信息

6. 符合法律法规
   - 严格遵守《个人信息保护法》相关规定
   - 收集信息与处理目的直接相关
   - 以明确、易懂的方式告知用户信息收集情况（隐私政策页面）
   - 用户享有知情权、删除权、投诉权

（二）防范违法有害信息传播扩展的技术措施

1. 信息传播控制
   - 用户只能向 AI 提问获取信息，**无法在网站内与其他用户互动**
   - **不存在用户生成内容（UGC）供他人查看的机制**
   - 网站本身**不提供站内转发、评论、私信等互动功能**
   - AI 问答页面提供"保存为图片"功能（客户端截图下载），
     用户可将 AI 回答截图保存到本地设备；该功能为客户端本
     地操作，网站不介入转发或传播环节，截图由用户在网站外
     自行处理
   - 所有 AI 回答均附带"AI 生成内容仅供参考"的免责标识，
     截图中也会保留该标识

2. 多重内容安全屏障
   - 输入过滤层：拦截恶意输入和注入攻击
   - AI Prompt 安全规则层：AI 拒绝生成违法违规内容
   - 内容标识层：明确标注 AI 生成内容性质，提醒用户理性判断

3. 访问限制
   - 密码验证限制 AI 功能访问范围
   - IP 速率限制防止 API 滥用
   - 暴力破解防护保障密码安全

4. 日志追溯
   - 完整记录所有交互过程
   - 发现违法信息时可追溯来源

（三）防范社会动员功能失控风险的技术措施

1. 本网站不具备社会动员功能基础
   - 无论坛、评论区、群组等用户互动功能
   - 用户之间无法互相交流或组织活动
   - 无用户发布/转发/分享功能
   - 无公告推送通知功能

2. 补充安全措施
   - 密码访问控制限制功能受众范围
   - 有明确的使用条款约束用户行为
   - 安全管理人员可随时关闭 AI 问答功能

3. 风险评估结论
   - 本网站社会动员功能失控风险等级：极低
   - 不存在形成网络组织、聚集、动员的机制和条件

- 投诉内容仅限安全管理负责人查阅和处理
